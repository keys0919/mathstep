# Development Rule

## 1. 단위
- 기능은 반드시 작은 단위로 분리
- 한 번에 하나의 기능만 구현

## 2. 구조
- 각 기능은 독립적으로 실행 가능해야 함
- 입력 / 출력이 명확해야 함

## 3. 순서
- 구현 → 테스트 → 다음 기능
- 중간 통합 금지

## 4. 출력 규칙
- 코드 우선
- 실행 가능한 상태로 제공
- 불필요한 설명 금지

## 5. Dependency
- package.json 수정 또는 패키지 설치 전후:
  Read and follow → ~/.claude/skills/expo-init.md
